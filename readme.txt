Тестовое задание было написано на PHP Version 5.4.45 настроенного на openserver.
Почтовый клиент так же настроен на openserver через один из моих e-mail.
Базу данных MySQL создал на своем хостинге, доступ открыт со всех ip, 
можно посмотреть реализацию тут https://xray.beget.com/phpMyAdmin , пароли в connect.php.

Тестовый акк с правами админа:
e-mail(login): admin@admin.ua
pass: admin

Структура:
/img		                //папка куда будут загружаться фото профиля
/includes
-changepass.php				//обработчик ajax запросов на смену и сброс паролей
-connect.php				//файл соединения к бд
-createdb.php				//sql запросы для создания бд из файла
/css
-style.css					//стили, reset.css не использовал
/
avatar.gif					//картинка по умолчанию
index.php					//главная страница, авторизация и т.д.
profile.php					//профиль пользователя

Что умеет делать реализация тестового задания:
1. Авторизация по e-mail;
2. Регистрация нового пользователя;
3. Сброс пароля, генерация нового и отправление его на почту;
4. Добавление и изменение фото профиля;
5. Изменение всех данных пользователя кроме e-mail;
6. Администратор может менять все поля включая e-mail (флаг);
7. Отображение даты и времени последнего посещения пользователя;
8. Изменение пароля с отправлением "напоминания" на e-mail;

Для того чтоб дать пользователю права админа, нужно в таблице users поменять roleid на 1.

Что не реализовано:
1. Не сделал валидацию форм:
- Нет ограничений на кол-во символов в пароле;
- Размер картинок.
2. Не сделал вывод различных ошибок:
- Если пользователь с таким e-mail уже существует;
- Не правильно введен e-mail при сбросе пароля;
- Пароли не совпадают / не правильно введен старый пароль.
3. Если админ меняет пароль - не приходит оповещение на новый и старый e-mail.

Потратил времени: 2 часа вечера в пятницу, 2 часа вечера субботы, 4 часа воскресения (2 из которых на тесты и описание readme.txt).
